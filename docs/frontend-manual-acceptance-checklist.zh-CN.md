# 新前端浏览器人工验收清单

本文档用于对新的前后端分离前端做发布前人工验收。

使用方式：

- 启动后端与新前端
- 按页面顺序逐项点击
- 对每一项标记为 `通过 / 不通过 / 暂不验证`
- 把“不通过”的项目整理成待修复问题

## 当前联调地址

- 后端：`http://127.0.0.1:4567`
- 前端：`http://127.0.0.1:5174`

## 验收结果标记建议

- `通过`
- `不通过`
- `暂不验证`
- `需要真实数据`

## 一、首页 `/`

### 数据加载

- [ ] 页面能正常打开，没有白屏或报错
- [ ] 数据库数量能显示
- [ ] 最近 BLAST 任务能显示
- [ ] 最近数据库索引任务能显示
- [ ] 最近数据库列表能显示

### 导航

- [ ] 点击“进入数据库管理”能进入 `/databases`
- [ ] 点击“进入 BLAST 提交”能进入 `/blast/new`
- [ ] 点击“进入任务中心”能进入 `/jobs`

## 二、数据库管理 `/databases`

### 页面加载

- [ ] 页面能正常打开
- [ ] 数据库列表能显示已有数据库
- [ ] 单个数据库的标题、类型、序列数、字符数能显示

### 直接输入 FASTA

- [ ] 选择“直接输入序列”模式
- [ ] 输入目标文件名后可提交
- [ ] 输入 FASTA 文本后可提交
- [ ] 勾选“自动建索引”后，提交成功返回任务信息
- [ ] 新建数据库能出现在数据库列表中
- [ ] 自动索引任务链接可点击

### 文件上传

- [ ] 选择“上传文件”模式
- [ ] 选择本地 FASTA 文件
- [ ] 提交成功
- [ ] 新数据库能出现在列表中

### 本机路径导入

- [ ] 选择“本机路径”模式
- [ ] 输入白名单路径中的 FASTA 文件
- [ ] 提交成功
- [ ] 当路径不在白名单中时，页面能显示明确错误

### 远程 URL 导入

- [ ] 选择“远程 URL”模式
- [ ] 输入允许的 URL
- [ ] 提交成功
- [ ] 当 URL 不在白名单中时，页面能显示明确错误

### S3 导入

- [ ] 选择“S3 地址”模式
- [ ] 输入允许的 `s3://` 或预签名 URL
- [ ] 提交成功
- [ ] 当 bucket 或 URL 不在白名单中时，页面能显示明确错误

### 索引与删除

- [ ] 点击“建立索引”可成功创建数据库索引任务
- [ ] 点击“删除数据库”后，数据库可从列表中消失
- [ ] 删除失败时能显示明确错误
