# 独立前端上线检查清单（中文）

本文用于在你准备把当前前后端分离版本公开部署或发布到 GitHub 前，快速检查关键项。

## 1. 代码与构建

- [ ] 新前端 `npm run build` 通过
- [ ] 新前端 `npm run test` 通过
- [ ] 后端 API 相关测试通过
- [ ] `openapi/api.yaml` 与当前接口实现没有明显脱节

## 2. 前端环境变量

- [ ] 已配置正确的 `VITE_API_BASE_URL`
- [ ] 开发环境与生产环境变量已区分
- [ ] 生产环境不再指向 `127.0.0.1:4567`

## 3. 后端跨域配置

- [ ] `~/.sequenceserver.conf` 已配置 `allowed_origins`
- [ ] 已包含正式前端域名
- [ ] 如需本地联调，也保留了本地开发地址

## 4. 外部导入安全

- [ ] 已明确是否开放 `local_path`
- [ ] 已明确是否开放 `url`
- [ ] 已明确是否开放 `s3`
- [ ] 如要开放，已配置最小白名单，而不是过大范围

相关配置：

- `allowed_import_paths`
- `allowed_import_urls`
- `allowed_s3_buckets`

## 5. 前端主流程回归

- [ ] 首页能正常加载数据库和任务概览
- [ ] 数据库页能查看当前数据库
- [ ] 数据库页能完成至少一种导入方式
- [ ] 数据库页能触发索引任务
- [ ] BLAST 提交页能成功创建任务
- [ ] 任务中心能看到新任务
- [ ] BLAST 详情页能查看状态、日志、结果
- [ ] 数据库任务详情页能查看状态、日志、结果

## 6. 失败路径检查

- [ ] 后端未启动时，前端能给出明确报错
- [ ] 白名单未配置时，外部导入能返回可理解错误
- [ ] BLAST 任务失败时，详情页能看到 stderr
- [ ] 索引任务失败时，详情页能看到 stderr

## 7. 部署与路由

- [ ] 静态文件目录已正确部署
- [ ] React Router 刷新不会返回 404
- [ ] `/api/v1/*` 已正确代理到后端
- [ ] Nginx / 反向代理未错误改写 API 路径

## 8. 文档

- [ ] `sequenceserver-web/README.md` 已同步到当前状态
- [ ] 部署说明已经补齐
- [ ] 白名单说明已经补齐
- [ ] GitHub 开源说明中已写清当前项目仍处于持续增强阶段

## 9. 开源发布前建议

- [ ] LICENSE 已保留并清楚
- [ ] README 中已说明与上游关系
- [ ] 已说明当前新增的独立前端目录和用途
- [ ] 已说明哪些功能已完成，哪些仍在开发

## 10. 当前最值得继续增强的方向

如果以上检查都完成，下一阶段最值得投入的通常是：

- 更完整的 BLAST 结果可视化
- 下载 / 导出
- 生产部署脚本或容器化
- 鉴权与权限控制
